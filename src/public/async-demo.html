Understanding the Event Loop

  • Synchronous (GREEN): Executes immediately, blocks the thread
  • Microtask/Promise (PURPLE): Queued in microtask queue, runs after current script but before macrotasks
  • setTimeout (RED): Queued in macrotask queue, runs after microtasks
  • Promise (BLUE): Creates a promise that may resolve synchronously or asynchronously
  • Async/Await (ORANGE): Syntactic sugar over promises

Demo Code Examples

Demo 1: Basic Execution Order

console.log('1. Synchronous start'); setTimeout(() => { console.log('2. setTimeout 0ms'); }, 0); Promise.resolve().then(() => { console.log('3. Promise.then'); }); console.log('4. Synchronous end'); // Order: 1 → 4 → 3 → 2

Demo 2: Promise Chain

Promise.resolve('Step 1') .then(result => { console.log(result); return 'Step 2'; }) .then(result => { console.log(result); });

Demo 3: Async/Await

async function demo() { console.log('Start async'); await Promise.resolve(); console.log('After await'); const result = await delay(1000); console.log(result); }
